# Codex Game Studios

The goal of this repository is to let Codex directly consume the game workflows originally designed for Claude Code, instead of rewriting a parallel system.

## Where to Start

1. Read `.agentlens/INDEX.md` first for navigation.
2. Repository-level rules are defined in this file.
3. The canonical source for specific workflows, skills, roles, and templates remains under `.claude/`.
4. The Codex adapter layer lives under `.agents/`; `.agents/skills/*/SKILL.md` is generated by script.

## Canonical Source

- Skill content is canonical in `.claude/skills/*/SKILL.md`.
- Role definitions are canonical in `.claude/agents/*.md`.
- Rules, templates, and process documentation are canonical in `.claude/docs/*` and `.claude/rules/*`.
- `.agents/skills/*/SKILL.md` is the adapter for Codex; do not edit it by hand. After changing `.claude/skills/*/SKILL.md`, run `python3 tools/sync_codex_adapters.py`.

## Codex Adapter Conventions

- When an original skill uses `AskUserQuestion`, in Codex ask the user directly only when truly blocked; otherwise continue with the narrowest, safest interpretation.
- When an original skill uses `Task` / subagent, use `spawn_agent` only if the user explicitly requested parallel work or delegation; otherwise execute locally.
- When an original skill uses `Write` / `Edit`, use `apply_patch` for manual file edits.
- When an original skill uses `WebSearch`, prefer `web` or Context7, and prioritize official or primary documentation.
- `.claude/docs/*`, templates, and rules referenced by original skills remain authoritative until a Codex-specific replacement exists.

## Change Boundaries

- This repository is already the Codex fork version; prefer placing new changes in `AGENTS.md`, `.agentlens/`, `.agents/`, `README.md`, `UPGRADING.md`, and necessary sync scripts.
- Unless the task directly requires it, do not rewrite `.claude/skills/*` workflow content in bulk. Make only necessary compatibility notes or source fixes.
- If skill content must be changed, update `.claude/skills/*/SKILL.md` first, then resync `.agents/skills/*`.

## Pre-Commit Checks

- `python3 tools/sync_codex_adapters.py --check`
- `git diff --check`
- When documentation mentions “today”, “recent”, or “current version”, prefer absolute dates, for example `2026-04-17`.
