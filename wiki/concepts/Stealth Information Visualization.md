# Stealth Information Visualization

Design pattern from [[Mark of the Ninja]] (Klei, 2012): make hidden game state **explicitly visible on-screen** to eliminate analog ambiguity. Cross-genre principle — applicable wherever a game has invisible enemy AI state (perception, awareness, predicted action).

## Klei's Five Visualizations

| Indicator | Encodes |
|---|---|
| Pulsing yellow cone | Guard's vision field — exact angle and reach |
| Cone color shift | Awareness level (calm → alerted → searching) |
| Blue ring around player | Noise radius generated by current action (sprint, drop, attack) |
| Smaller blue ring | Quiet noise — walking, breathing |
| Outline glow on guard dog | Smell radius (animals' alternate sense) |

## Binary State Principle

> [!key-insight] Klei's design rule
> **Stealth is binary, not analog.** Either the player is **hidden** or **illuminated**. No "partial cover" / light-gem-percentage mid-state. The player always knows exactly which of the two states they're in. Decisions follow from clarity.

This contradicts pre-2012 stealth games (Splinter Cell light gem 0–100, Hitman partial-suspicion meters) which used analog states and led to "trial and error" frustration.

## Cross-Genre Applications

### Run-and-Gun (Echo specifically)

Time-rewind is **invisible state**. Player has tokens; bullets had paths; rewinding repaints the past. Borrow Klei's principle:

- **Token count**: explicit HUD counter (binary "have token / don't have token")
- **Predicted bullet trails**: when rewind charge is held, render red ghost-trails of bullets that **will** hit on the un-rewound timeline. Essentially Mark-of-the-Ninja's vision cones for the future.
- **Rewindable window**: silver afterimage of player's path over the last N seconds — visualize the rewindable interval explicitly.

### Other Genres

- **Souls-like**: explicit poise meter overlay (currently hidden)
- **Roguelite**: explicit drop-rate display on item rarity (Risk of Rain 2 uses this)
- **Run-and-gun**: parry/i-frame windows shown as colored tint on character sprite

## Implementation Cost (low)

UI overlay only — no new gameplay logic. Render bullets twice (once normal, once predicted-red). Render player path as silver afterimage shader. Rewindable interval as semi-transparent strip at top of HUD. All cosmetic; can be toggled off as accessibility option.

## Echo Tier 1 Recommendation

> [!key-insight] Top cross-genre transplant for Echo
> Adopt Klei's visualization principle for the time-rewind mechanic. Show the predicted bullet trails, the token count, the rewindable window — all explicitly. **Cheapest cross-genre transplant with the highest synergy with the locked differentiator.** Implement alongside the time-rewind v0 prototype (1-week target per [[Solo Contra 2026 Concept]]).
