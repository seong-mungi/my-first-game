# Run and Gun Extension Systems

## 개념 정의

**런앤건 확장 시스템(Run and Gun Extension Systems)**은 [[Run and Gun Base Systems]]의 5가지 핵심 위에 성공적으로 접목된 설계 패턴들이다.

확장 시스템은:
- 기반 시스템을 **대체하지 않고** 위에 쌓인다.
- 각각 고유한 재플레이성·깊이·시장 포지셔닝을 제공한다.
- 어떤 확장을 선택하느냐가 게임의 **장르 포지셔닝**을 결정한다.

## 확장 패턴 카탈로그

### Extension A: 탈것 / 슬러그 시스템 (Vehicle System)

**대표**: Metal Slug 시리즈 (SV-001, Slug Flyer, Slug Mariner 등)

**메커닉**: 특정 지점에서 탈것 탑승 → 화력+내구도 대폭 향상 → 조작 변화 → 탈것 파괴 후 귀환.

**설계 효과**:
- 스테이지 다양성: 지상/공중/수중 구간별 다른 조작감.
- 일시적 강화: "슬러그를 아껴야 한다" vs "전진 화력 자랑"의 긴장.
- 비주얼 임팩트: 거대 탈것이 화면을 채우는 스펙터클.

**구현 비용**: 중간. 탈것별 별도 애니메이션·충돌 로직 필요.

**접목 조건**: 탈것마다 "이 탈것이 최적인 구간"이 존재해야 한다. 아무 스테이지에서나 모두 최강이면 의미가 없다.

---

### Extension B: POW/NPC 구출 경제 (Rescue Economy)

**대표**: Metal Slug 시리즈 (POW 구출 → 아이템/점수)

**메커닉**: 숨겨진 NPC를 구출하면 보상. 사망 시 카운트 리셋 → 위험/보상 구조.

**설계 효과**:
- 탐험 동기: 선형 진행에 숨겨진 목표 추가.
- 위험 보상: 적 밀집 구역에서 포로를 구출하는 선택.
- 리플레이성: 모든 포로 구출 달성을 위한 반복 플레이.

**구현 비용**: 낮음. 숨겨진 NPC 배치 + 보상 로직.

---

### Extension C: 분기 경로 (Branching Paths)

**대표**: Metal Slug 3 (5개 루트 선택 가능)

**메커닉**: 스테이지 중 혹은 스테이지 시작 시 경로를 선택. 각 경로마다 다른 적·환경·비주얼.

**설계 효과**:
- 리플레이성 극대화: 한 번의 플레이로 모든 콘텐츠를 못 봄.
- 난이도 선택: 일부 경로가 더 어렵거나 더 많은 보상.
- 서사 다양성: 경로마다 다른 시각적 스토리텔링.

**구현 비용**: 높음. 각 분기마다 독립적인 콘텐츠 제작 필요.

**접목 조건**: 콘텐츠 생산량이 충분해야 한다. 소규모 팀에서는 2~3분기가 최대 현실적 범위.

---

### Extension D: 무기 조합 시스템 (Weapon Combination)

**대표**: Gunstar Heroes (4 기본 타입 × 조합 = 10+ 결과)

**메커닉**: 기본 무기 타입을 조합해 새로운 무기 생성. 빌드 선택 레이어 추가.

**설계 효과**:
- 빌드 다양성: 동일 기본 요소로 전혀 다른 플레이스타일.
- 발견의 즐거움: 조합 결과를 탐색하는 과정.
- 전략 깊이: "이 스테이지엔 어떤 조합이 최적?" 선택.

**구현 비용**: 중간~높음. 조합 결과물을 모두 밸런싱해야 함.

---

### Extension E: 보스 러시 하이브리드 (Boss Rush Core)

**대표**: Cuphead (2017)

**메커닉**: 런앤건 런 스테이지를 보조로, 보스 전투를 주요 콘텐츠로 전환.

**설계 효과**:
- 보스 디자인의 심화: 페이즈별 패턴 복잡도 극대화.
- 짧은 런 스테이지: 낮은 제작 비용으로 장르 정체성 유지.
- 높은 재플레이성: 보스 하나를 완벽히 클리어하기 위한 반복.

**구현 비용**: 보스 디자인에 대부분의 자원 집중. 런 스테이지 비중 낮음.

**접목 조건**: 보스 전투 디자인 역량이 있어야 한다. 보스가 재미없으면 게임 전체가 실패.

---

### Extension F: 로그라이트 진행 (Roguelite Progression)

**대표**: Huntdown: Overtime (2026 Early Access), 개념적으로 다수 인디

**메커닉**:
- 런 실패 시 부분 리셋 (아이템/업그레이드 일부 유지).
- 런마다 랜덤 무기·파워업 조합.
- 메타 업그레이드로 캐릭터/빌드 성장.

**설계 효과**:
- 리플레이성 구조화: 각 런이 다른 경험.
- 진행 허들 완화: "이번엔 못 깨도 영구 성장."
- 현대 시장 포지셔닝: 로그라이트는 현재 인디 시장 주류 장르.

**구현 비용**: 높음. 랜덤 생성 시스템 + 메타 진행 + 밸런싱.

**접목 위험**: 런앤건의 "패턴 암기" 철학과 로그라이트의 "랜덤 대응" 철학이 충돌할 수 있음.

> [!gap] 로그라이트 런앤건 성공 사례가 아직 적음. Huntdown: Overtime은 2026년 5월 Early Access 진입. 장기 성공 여부 미확인.

---

### Extension G: 파괴 가능 환경 (Destructible Environment)

**대표**: Broforce (2015)

**메커닉**: 모든 지형이 총탄/폭발로 파괴 가능. 전략적 지형 파괴 + 붕괴 위험.

**설계 효과**:
- 전술 창발성: 지형을 무기로 사용, 적 위의 천장 붕괴.
- 매 런 다른 지형: 이전 플레이의 파괴가 현재 상태에 영향.
- 스펙터클: 폭발 연쇄가 화면을 채우는 시각 효과.

**구현 비용**: 높음. 복셀/타일 파괴 물리 시뮬레이션.

---

### Extension H: 캐릭터 로스터 전환 (Character Roster Switching)

**대표**: Broforce (구출한 포로 = 새 캐릭터로 전환), Metal Slug X+ (4캐릭터 선택)

**메커닉**: 구출/획득한 캐릭터로 전환. 각 캐릭터마다 고유 무기·능력.

**설계 효과**:
- 탐험 동기: 새 캐릭터를 위한 구출 행동.
- 다양한 플레이스타일: 캐릭터마다 다른 조작감.
- 수집 욕구: 모든 캐릭터 잠금 해제 목표.

**구현 비용**: 중간. 캐릭터별 애니메이션·스탯·무기 세트.

---

### Extension I: RPG 진행 (RPG Progression)

**대표**: Mercenary Kings (2014) — 총기 제작 + 스탯 업그레이드

**메커닉**:
- 미션 클리어 → 재료 획득 → 무기 제작/강화.
- 플레이어 캐릭터 스탯 성장.
- 퀘스트 구조.

**설계 효과**:
- 장기 세션 목표: 단순 스테이지 클리어 이상의 목표.
- 커스터마이제이션: 자신만의 빌드.
- 콘텐츠 밀도: 동일 스테이지를 여러 번 플레이하게 만드는 구조.

**구현 비용**: 높음. 아이템 시스템 + 밸런싱 전체.

**접목 위험**: 런앤건 특유의 액션 스피드가 느려질 수 있음.

## Implications For my-game

런앤건 + 확장 시스템 선택 권고:

**소규모 팀 (인디 3~5인)의 현실적 조합:**

```
기반 (필수):
  이동 + 점프 + 사격 + 적 패턴 + 피격 결과

1차 확장 (v1에 포함 권장):
  + 무기 픽업 시스템 (3~5종, [[Weapon Letter Pickup System]])
  + 보스 러시 구조 (각 스테이지 끝 보스, 복수 페이즈)
  + 2인 로컬 협동

2차 확장 (v1.5 또는 DLC):
  + 탈것 1종 (비용 대비 임팩트 높음)
  + 분기 경로 1개 (2~3스테이지에서 선택)

장기 확장 (v2 또는 후속작):
  + 로그라이트 진행 (별도 모드로)
  + 캐릭터 로스터 추가
```

**핵심 결정**: 로그라이트를 도입하면 시장 적합성은 높아지지만 장르 정체성이 흐려진다. "순수 런앤건"과 "로그라이트 런앤건" 중 하나를 먼저 선택하고 나머지를 설계해야 한다.

## Open Questions

- 보스 러시(Cuphead 방식)와 전통 스테이지 구조(Metal Slug 방식) 중 소규모 팀에게 더 유리한 것은?
- 로그라이트 확장이 "런앤건 팬"을 놓치고 "로그라이트 팬"을 얻는 트레이드오프를 감수할 가치가 있는가?
- my-game의 핵심 플레이어가 누구인지(레트로 장르 팬 vs 현대 인디 게이머)에 따라 확장 선택이 달라진다.
