# 코나미 코드 (Konami Code)

## 기본 정보

**입력 시퀀스:** ↑ ↑ ↓ ↓ ← → ← → B A (+ Start 또는 Select)
**최초 창안자:** 하시모토 카즈히사(Kazuhisa Hashimoto), 코나미 프로그래머
**첫 게임 삽입:** Gradius (NES, 1986)
**대중화:** [[Contra]] (NES, 1988) — "30 lives" 코드로 북미 전역에 알려짐

## 탄생 배경

### 개발 목적: 테스트 편의
하시모토는 1985년 아케이드 Gradius를 NES로 포팅하는 작업 중 "테스트 중에 직접 플레이하기 너무 어렵다"는 문제에 직면했다. 게임의 최초 스테이지부터 완전 파워업 상태로 시작하는 치트를 삽입해 테스트 효율을 높였다.

### 삭제되지 않은 이유
대량 생산 준비 단계에서 코드의 존재가 발견됐으나, **새 버그 도입 위험 때문에 제거를 포기**했다. 결과적으로 코드는 그대로 최종 제품에 포함됐다.

**시퀀스 선택의 논리:**
- 테스터가 기억하기 충분히 쉬움
- 게임 플레이 중 실수로 입력할 가능성이 충분히 낮음
- 패드의 십자키를 체계적으로 활용하는 패턴

## Contra에서의 역할

### NES Contra (1988)의 30 Lives 코드
- **입력 위치:** 타이틀 화면에서 1P 컨트롤러로 입력
- **효과:** 시작 잔기 3 → **30 목숨**으로 증가
- **의미:** NES Contra는 1히트 즉사 + 잔기 3의 극악 난이도 게임. 30 lives가 없으면 대부분의 플레이어가 1-2스테이지에서 게임 오버. → 코드가 사실상 **접근성 패치** 역할

### 비공식 명칭
이 코드는 북미에서 "Contra Code" 또는 "30 Lives Code"로 불렸다. Contra가 코드의 대중화에 기여한 사례로, 원래 Gradius 코드임에도 Contra와의 연결이 더 강하게 인식됐다.

> [!gap] 코드가 Contra 판매에 미친 영향(접근성 향상 → 판매 증가 vs 클리어율 증가 → 재판매가) 관련 정량 데이터 없음.

## 후속 게임들에서의 사용

코나미 코드는 Contra 이후 코나미의 다수 게임에 포함됐다:
- **Gradius 시리즈** — 완전 파워업
- **Castlevania** 시리즈 일부 — 스킵/보너스
- **Dance Dance Revolution** — 숨겨진 메뉴
- 기타 코나미 자사 타이틀 다수

코나미 외부 회사들도 자사 게임에 이스터에그로 동일 시퀀스를 삽입하기 시작했다.

## 게임 외부로의 확산

### 웹 & 디지털 플랫폼
입력 가능한 현대 서비스들:
| 플랫폼 | 효과 |
|---|---|
| Netflix | 파티 이스터에그 |
| Discord | 특수 이모지/효과 |
| Twitter/X | 이스터에그 |
| Google Assistant / Siri / Alexa | 음성 인식 반응 |
| ESPN, Facebook | 숨겨진 UI 효과 |
| Bank of Canada | 공식 웹사이트 이스터에그 |

### 대중문화
- **영화 Wreck-It Ralph (2012):** 코나미 코드가 직접 등장 — 3세대 비디오게임 문화를 상징하는 기호로 사용됨
- 게임 관련 어떤 이스터에그보다도 광범위하게 인용되는 게임 문화 레퍼런스

## 설계적 함의

### "기억 가능한 패턴"의 힘
코나미 코드가 40년간 살아남은 이유는 **구조적 기억 가능성**이다:
- ↑↑↓↓: 위아래 반복 (대칭 패턴)
- ←→←→: 좌우 반복 (대칭 패턴)
- BA: 두 버튼 (종결 패턴)

세 부분으로 나뉜 청크 구조가 암기를 쉽게 한다. 이는 UI 디자인의 "청킹(chunking)" 원칙의 자연 적용 사례다.

### "의도된 접근성"의 역사적 맥락
아케이드 [[Arcade Difficulty Design]]에서 1히트 즉사 + 잔기 제한은 코인 수익 최대화를 위한 의도된 설계였다. 코나미 코드는 이 구조를 **콘솔 환경에서 유연화**하는 방법으로 기능했다. "어렵지만 코드를 쓰면 즐길 수 있다"는 이중 구조.

## Implications For my-game

1. **치트/이스터에그는 커뮤니티 형성 자산**: 코나미 코드가 "비밀을 공유하는 커뮤니티 경험"을 만들었다. my-game에 숨겨진 시퀀스를 넣는 것은 바이럴 요소가 될 수 있다.
2. **접근성의 이중 레이어**: "어렵지만 코드로 완화 가능" 구조는 핵심 플레이어와 캐주얼 플레이어를 동시에 만족시키는 고전 해법. 현대 게임의 난이도 설정과 다른 "발견의 재미"를 포함.
3. **기억하기 쉬운 입력 패턴**: 아케이드/콘솔 UI에서 입력 시퀀스를 요구할 때 청크 구조로 설계하면 사용자 기억 부담 최소화.
