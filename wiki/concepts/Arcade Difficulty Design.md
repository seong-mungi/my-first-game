# Arcade Difficulty Design

## 패턴 정의

**아케이드 난이도 설계(Arcade Difficulty Design)**는 코인 투입 경제를 최적화하기 위해 발전한 난이도 철학이다. 런앤건을 포함한 대부분의 아케이드 게임에서 표준으로 자리잡았다.

핵심 원칙: **"충분히 어려워서 코인을 계속 투입하게, 충분히 재밌어서 다시 투입하게."**

## 아케이드 경제 메커닉

### 코인 피드 설계 (Quarter-Feed Design)
- 아케이드 기체 소유자(오퍼레이터)의 목표: 수익 극대화.
- 설계 목표: 플레이어가 2~3분마다 코인을 투입하도록.
- 이상적 아케이드 게임: 실력자는 1코인으로 클리어 **가능**하지만, 평균 플레이어는 수십 코인이 필요.
- 보통 게이머가 전체 엔딩을 보려면 수주~수개월 반복 플레이 + 다수 코인 필요.

### 1히트 킬 (One-Hit Kill)
**런앤건 아케이드의 핵심 난이도 레버.**

- 총알 1발, 폭발 1회, 적 접촉 1번 = 즉사.
- 예외: 탈것(Metal Slug 슬러그) — 내구도로 대체.
- 효과:
  - 게임 속도를 극적으로 높임 (항상 주의 집중)
  - 실수 비용을 최대화 (즉시 컨티뉴 압력)
  - 패턴 암기의 중요성 극대화

> [!gap] "1히트 킬"과 "체력바" 중 어느 것이 더 재미있는가는 여전히 논쟁적. 체력바는 실수 완충이지만, 1히트 킬은 집중도와 긴장감이 더 높다는 주장이 있음.

### 목숨과 컨티뉴 시스템

| 레이어 | 기능 | 리셋 단위 |
|---|---|---|
| 무적 시간(i-frame) | 피격 직후 잠시 무적 | 없음 (1히트 = 즉사) |
| 목숨(Lives) | 사망 후 재시작 횟수. 기본 3~5개 | 컨티뉴 시 일부 보충 |
| 컨티뉴(Continue) | 코인 투입으로 계속 플레이 | 스테이지 중간 또는 스테이지 처음부터 |
| 게임 오버 | 컨티뉴 소진 시. 처음부터 | 전체 리셋 |

**Metal Slug 특수 규칙**: 사망 시 POW 카운트 0으로 초기화 — 추가 패널티 레이어.

### 패턴 암기 (Pattern Learning)

아케이드 난이도의 본질: **"처벌→암기→성장→다음 처벌"의 반복 루프.**

- 적과 보스의 공격은 결정론적 패턴 (랜덤 없음).
- 죽음은 정보다: "다음엔 이 패턴을 피할 수 있다."
- 충분히 암기하면 무손실(No-Damage) 클리어 가능.
- 이것이 아케이드 난이도와 "불공정한 난이도"의 차이.

## 현대적 재해석

### Cuphead의 아케이드 철학 계승

Cuphead(2017)는 아케이드 난이도를 현대 콘솔/PC 게임에 이식한 성공 사례:
- 1히트 킬 유지 (보스 전투).
- 대신 **무제한 재시도** — 코인 경제 제거. 패턴 학습에 집중.
- 여러 난이도 옵션(Simple/Regular/Expert) — 접근성 배려.
- 결과: "극한 난이도"가 마케팅 포인트로 기능.

### 아케이드 vs 현대 콘솔 설계 비교

| 요소 | 아케이드 원형 | 현대 콘솔 적용 |
|---|---|---|
| 실패 비용 | 코인 투입 (현금) | 시간 낭비 |
| 컨티뉴 | 유한 (코인당 1개) | 무한 재시도 |
| 체크포인트 | 없거나 스테이지 단위 | 스테이지 내 중간 체크포인트 |
| 난이도 선택 | 오퍼레이터 설정 | 플레이어 선택 |
| 목표 | 코인 투입 유도 | 완주 경험 제공 |

### 난이도 커뮤니케이션 설계

현대 런앤건 인디가 채택하는 방식:
- **"공정한 어려움"**: 모든 죽음이 학습 가능한 이유 있음. 플레이어가 속았다고 느끼지 않아야.
- **시각적 경고 신호**: 공격 전 예고 동작(Telegraph) 충분히 제공.
- **즉각적 재시작**: 로딩 없이 0.5초 내 재시작 → 리트라이 마찰 최소화.
- **진행 피드백**: "이번엔 2페이즈까지 갔다" — 성장 감각 제공.

## 난이도 설계의 핵심 레버

런앤건 난이도를 조절하는 주요 변수:

| 레버 | 쉬운 방향 | 어려운 방향 |
|---|---|---|
| 플레이어 내구도 | 체력바 (3~5히트) | 1히트 킬 |
| 적 탄환 속도 | 느림, 예측 가능 | 빠름, 복잡한 패턴 |
| 적 배치 밀도 | 희소, 간격 큼 | 밀집, 동시 공격 |
| 체크포인트 위치 | 잦은 중간 저장 | 스테이지 시작부터 |
| 컨티뉴 횟수 | 무제한 | 0~3회 |
| 보스 페이즈 수 | 1~2페이즈 | 3~6페이즈 |
| 텔레그래프 시간 | 길고 명확한 예고 | 짧고 불명확한 예고 |

## Implications For my-game

1. **1히트 킬 채택 여부 결정**: 런앤건 정체성의 핵심 선택. 1히트 킬은 긴장감 극대화이지만, 캐주얼 시장 접근성을 좁힌다. 체력바(2~3히트)는 현대 관객 친화적이지만 장르 고유 긴장감이 줄어든다. **Cuphead 해법**: 1히트 킬 + 무제한 재시도 + 난이도 선택 옵션.
2. **즉각 재시작이 필수**: 죽음 → 재시작 간격이 1초 이상이면 리트라이 동기가 급격히 떨어진다. 로딩 없는 즉각 재시작 구현이 난이도 설계만큼 중요.
3. **공정성의 시각화**: 모든 적 공격에 텔레그래프(예고 동작)를 설계. 플레이어가 "피할 수 있었다"고 느껴야 다시 시도한다.
4. **난이도 옵션**: 코어 게이머(1히트 킬)와 캐주얼 게이머(체력바) 모두를 위한 옵션. 메인 경험은 코어 설계 기준, Easy 모드는 체력바 추가.
5. **아케이드 경제 없이 아케이드 긴장감 유지**: 코인 없이도 긴장감을 만들려면 "진행 리셋의 공포"가 필요. 체크포인트 위치와 재시작 단위가 핵심.
