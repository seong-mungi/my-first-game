# 런앤건 협동 플레이 설계

## 핵심 구분: 진정한 동시 플레이 vs 양보형 시뮬레이션

런앤건의 협동 플레이는 크게 두 유형으로 나뉜다.

```
┌─────────────────────────────────────────────────────────────┐
│  진정한 동시 플레이 (True Simultaneous Co-op)                │
│  두 플레이어가 동시에, 독립적으로, 같은 화면에서 행동        │
│  예: Contra, Metal Slug, Broforce                           │
├─────────────────────────────────────────────────────────────┤
│  양보형 시뮬레이션 (Turn-Based / Handoff Co-op)             │
│  한 플레이어가 죽을 때 다른 플레이어가 이어받는 방식        │
│  또는 한 명이 주도하고 나머지가 보조                        │
│  예: 일부 모바일 런앤건, 싱글플레이 위주 변형                │
└─────────────────────────────────────────────────────────────┘
```

Contra(1987)가 확립한 **진정한 동시 플레이** 모델이 런앤건 장르의 표준이다.

## 진정한 동시 플레이의 설계 원칙

### 1. 독립 상태 공유 화면
- 두 플레이어가 **별도의 목숨, 별도의 무기, 별도의 입력**을 가짐
- 그러나 **같은 화면** 안에서 진행 — 화면 분할(split-screen) 없음
- 한 플레이어의 사망이 다른 플레이어의 진행을 중단시키지 않음

**화면 공유의 함의:**
- 한 플레이어가 앞서 나가면 다른 플레이어가 화면 뒤로 밀릴 수 있음 (아케이드 스크롤 한계)
- 두 플레이어가 서로의 시야를 확인하고 위치를 조율해야 함
- 공동 생존 vs 개인 생존 사이의 긴장감 발생

### 2. 협동의 구체적 이점
단순히 "둘이서 한다"는 것 이상의 구체적 협동 메커니즘:

| 협동 요소 | Contra | Metal Slug | Broforce |
|---|---|---|---|
| 화력 집중 | 두 플레이어 동시 보스 공격 | 동일 | 4인 집중 가능 |
| 위기 커버 | 한 명 사망 후 재등장 시 다른 명이 공간 유지 | 동일 | 프렌들리 파이어 위험 추가 |
| 탐색 분업 | 이론적(아케이드에선 제한) | 분기 경로에서 유효(MS3) | 파괴 환경에서 경로 분기 |
| 심리 지원 | "같이 있다"는 경험 자체 | 동일 | 커뮤니케이션 더 필요 |

### 3. 동시 플레이가 난이도에 미치는 효과
2인 협동은 보통 난이도를 낮추는 것으로 보이지만 실제로는 복합적:

**난이도 감소 요소:**
- 화력 2배 → 보스/적 처리 속도 향상
- 한 명 사망 시 다른 명이 버퍼 역할
- 주의 분산 — 한 명이 상단, 한 명이 하단 커버

**난이도 증가 요소:**
- 화면이 혼잡해져 탄환/적 식별 어려움
- 두 플레이어의 진행 속도 불일치 → 화면 스크롤 충돌
- 무기 픽업 경쟁 (Contra에서 1명이 모든 S샷 독점 가능)
- Broforce의 프렌들리 파이어 → 실수 비용 증가

**결론:** 잘 설계된 런앤건 협동은 **쉬워지는 것이 아니라 "다른 종류의 어려움"으로 전환**된다.

## 주요 게임별 협동 설계 비교

### Contra (1987) — 협동의 원형

**구조:**
- 2인 동시, 같은 화면, 아케이드
- Bill(파란 바지) + Lance(빨간 바지) — 외형 차이만, 능력 동일
- 공유 요소: 화면, 스크롤 진행, 보스
- 독립 요소: 목숨, 무기, 사망

**핵심 특성:**
- "1인 플레이 콘텐츠를 2인이 함께 통과" 구조 — 레벨은 1인용으로 설계, 협동으로 완화
- 아케이드 경제: 2인이 코인을 각자 투입 → 수익 2배
- 협동이 아케이드 수익 모델과 완벽하게 정렬됨 (Source: [[Wikipedia Run and Gun]])

**게임 인기에서 협동의 기여:**
> "Much of the game's popularity came from its two-player simultaneous co-op gameplay."
> (Source: Wikipedia Contra)

### Metal Slug (1996) — 협동 + 탐색 확장

**구조:**
- 2인 동시 (Marco + Tarma, 이후 Eri + Fio 추가)
- 능력 동일 — 차이는 외형/애니메이션
- Metal Slug 3부터 분기 경로: 두 플레이어가 다른 경로 탐색 가능

**Contra와의 차이:**
- POW 구출 시스템이 협동에 탐색 레이어 추가 — 두 명이 다른 경로를 맡을 때 POW 발견 기회 증가
- 탈것(슬러그) 공유: 한 명 탑승 + 한 명 도보 조합 전술
- Metal Slug 6의 3무기 캐리 시스템: 협동 상황에서 무기 보유 전략 다양화

### Broforce (2015) — 협동의 카오스 극대화

**구조:**
- **최대 4인** 동시, 로컬 + 온라인
- **프렌들리 파이어 활성화** — 아군 공격 가능
- **캐릭터 셔플**: 포로 구출 시 무작위 캐릭터 교체 → 각 플레이어가 다른 캐릭터/무기 보유
- **파괴 가능 환경**: 아군의 폭발이 지형을 파괴 → 의도치 않은 협동/방해

**협동 설계의 철학적 전환:**
- Contra/Metal Slug: 협동 = 목표 달성을 위한 전략적 협력
- Broforce: 협동 = **예측 불가능한 혼란 그 자체가 오락**
- 프렌들리 파이어가 "실수의 재미"와 "상황 코미디"를 생성

### Cuphead (2017) — 협동의 절제

**구조:**
- 2인 로컬 협동 (Cuphead + Mugman)
- 온라인 협동 없음 (일부 비판)
- 플레이어 1이 사망해도 플레이어 2가 일정 시간 내 부활 가능 (슈퍼 미터 소모)

**차별점:**
- 부활 시스템이 협동의 핵심 메커닉 — "파트너를 살리느냐 마느냐"의 결정
- 보스 러시 구조에서 협동: 두 플레이어가 각자 다른 페이즈/공격에 집중하는 암묵적 역할 분담

## 협동 설계의 핵심 의사결정

```
Q1. 몇 명까지 지원?
  └─ 2인: Contra/Metal Slug/Cuphead — 설계 단순, 화면 관리 용이
  └─ 4인: Broforce — 카오스 수용, 온라인 지원 필요

Q2. 캐릭터 능력 동일 vs 차별?
  └─ 동일: Contra/Metal Slug — 협동 전략은 위치/무기로만
  └─ 차별: Broforce (캐릭터 셔플), Cuphead (Mugman 색상만) — 역할 분담 가능

Q3. 프렌들리 파이어?
  └─ 없음: Contra/Metal Slug/Cuphead — 실수 비용 감소, 협동 진입 장벽 낮춤
  └─ 있음: Broforce — 긴장감 + 코미디 + 숙련도 요구

Q4. 로컬만 vs 온라인 지원?
  └─ 로컬만: Cuphead(출시 시점), Contra NES — 개발 단순
  └─ 온라인 포함: Broforce — 도달 범위 확대, 개발 복잡도 증가
```

## Implications For my-game

1. **2인 동시 협동부터**: Contra/Metal Slug가 증명한 최소 협동 단위. 4인은 선택이나 2인은 장르 정체성.
2. **프렌들리 파이어는 게임 톤에 맞춰 결정**: 진지한 액션 → OFF (Contra 방식). 파괴적 코미디 → ON (Broforce 방식).
3. **협동은 난이도 조절 도구**: 2인 협동 자체가 접근성 패치 역할. [[Arcade Difficulty Design]]의 극한 난이도도 협동으로 완화 가능.
4. **온라인보다 로컬 협동 먼저**: Cuphead가 로컬만으로 600만 장 판매. 온라인은 추후 추가 가능.
5. **화면 공유 UX**: 두 플레이어가 같은 화면에 있을 때의 혼잡도 관리 — 플레이어 식별 색상, 화면 경계 처리 필요.

- Contra 스타일 2인 협동은 솔로 플레이어에게 얼마나 불리한가? (레벨 설계가 협동 기준인가 솔로 기준인가)
